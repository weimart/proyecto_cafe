## 🏗️ DECISIONES TÉCNICAS CLAVE

### 1. FRONTEND SIN FRAMEWORKS PESADOS
**Decisión:** Vanilla JS + módulos simples en lugar de React/Vue
**Razón:** Reducir bundle size, mejorar LCP (< 2.5s)
**Beneficio:** 0 dependencias de framework, control total
**Alternativa SI necesitas complejidad:** React 18 + Vite

### 2. CSS VARIABLES EN :ROOT
**Decisión:** Sistema completo de variables CSS
**Razón:** Mantenimiento fácil, tema coherente
**Beneficio:** Cambiar color primario = 1 variable
**Ubicación:** 02_SISTEMA_CSS_AVANZADO.css (línea ~8)

### 3. SCHEMA.ORG INLINE EN HTML
**Decisión:** Schema.org JSON-LD directamente en <head>
**Razón:** Mejor SEO, Google entiende estructura
**Tipos:** LocalBusiness + Product + Breadcrumb
**Validar:** https://schema.org/validator/

### 4. NGINX CON 3 NIVELES DE CACHE
**Decisión:** Browser + CDN + Server cache
**Razón:** Máximo rendimiento, reducir carga BD
**Tiempos:** 1 año assets | 24h imágenes | 10m API
**Config:** 04_NGINX_PRODUCTION_CONFIG.conf

### 5. LAZY LOADING CON INTERSECTION OBSERVER
**Decisión:** Cargar imágenes/módulos bajo demanda
**Razón:** Mejorar FCP, reducir carga inicial
**Implementación:** 05_JAVASCRIPT_MODULAR.js (LazyLoader)

## 🚀 PERFORMANCE OPTIMIZATIONS

### Critical Path
```
1. Preload Google Fonts (woff2)           ← 200ms
2. Inline CSS crítico (above-the-fold)    ← 0ms
3. Defer scripts no-críticos               ← async/defer
4. Lazy load imágenes (Intersection)       ← on-demand
5. Minify assets (JS, CSS)                 ← -40% size
6. Gzip compression (Nginx lvl 6)          ← -70% size
```

### Caching Strategy
```
Browser Cache:   1 año     (assets estáticos)
CDN Cache:       24 horas  (imágenes)
Redis Cache:     10 minutos (API queries)
DB Indexes:      Optimizados (usuarios, productos)
```
